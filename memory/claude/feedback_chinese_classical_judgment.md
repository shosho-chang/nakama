「好生 X」是合法的副詞用法（=「很 X」/「真令人 X」的書面或古雅形式），不是「好深 X」的錯字。

**事件**：iter3 SRT 寫「就好生羨慕啊」、Memo 跟 iter4 SRT 寫「就好深羨慕啊」。我用 Memo 當 baseline 反推 iter3 ASR 錯成「好生」、iter4 換成 stable-ts 後「修對」成「好深」— 修修指正：**顛倒了**。「好生」才是更精準的台灣繁中文雅寫法，「好深」反而是同音相近的另一個合法但更口語的講法。

**Why**：Claude 中文語感偏現代普通話 + 簡中口語語料，台灣繁中保留更多文言副詞 / 書面詞 / 文雅句式（好生、甚是、俱已、方才、頗有、毫無、咱們、儘管 ...）。我容易把不熟悉的合法用法當 ASR 錯字。

**How to apply**：
- ASR 文字品質判斷時，**「不認得的詞」「奇怪的搭配」先當合法用法**，不立刻判錯
- baseline 對比（用 Memo / 其他 ASR）— **baseline 不是 ground truth**，只是另一份 ASR 輸出；不一致不代表哪邊錯，可能兩邊都對只是擇詞不同
- 真 ASR 錯誤要有強證據（音/形差距大如「諸位遊牧」vs「數位遊牧」、上下文邏輯矛盾如「跟友」vs「跟 Paul」）
- 邊界 case 直接問修修，不在報告中下定論說「iter X 修對 / 錯」
- 同款 risk pair：「好生 / 好深」、「甚是 / 甚為」、「俱已 / 都已」、「方才 / 剛才」、「頗有 / 很有」、「儘管 / 雖然」
