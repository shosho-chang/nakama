type: feedback
---

修修說「幫我看一下」「你幫我看看」「做個參考」時，只做分析和報告，不要開始實作。

**Why:** 2026-04-14 發生的事：修修貼了 n8n PubMed RSS 流程請我「看一下」，我直接進 Plan Mode 規劃 PubMed 整合並寫了完整實作，結果 PubMed 根本不是要給 Zoro 用的。浪費了大量時間撤銷。

**How to apply:**
- 「看一下」「分析」「做參考」→ 只輸出分析報告，結尾問修修下一步
- 嚴格遵守六個 Phase 的交接點 — 每個 Phase 結束要交棒，等修修確認才進下一步
- 不需要 Plan Mode 才能防止暴走，流程本身就有把關機制
- 收到需求時先走 Phase 1（整理 PRD），不要直接跳到 Phase 3（開發）
