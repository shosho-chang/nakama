name: 美學是 first-class concern
description: 自用工具（Bridge）和對外產出（文章、vault、社群 UI）都要求賞心悅目，不接受 AI slop default
type: feedback
---

修修對美學有要求：不管是自己每天用的工具（Bridge UI、Obsidian 呈現），還是對外產出給用戶用的介面（Brook 文章、Nami vault 頁面、未來 Chopper 社群），都要「看得賞心悅目、工作起來比較開心」。

**Why:** 長期 DX / UX 的投資觀點 — 賞心悅目的工具會讓使用者（含他自己）更願意回來用，也會讓產出品質跟著拉高；反之 AI slop 的預設樣貌（Inter + 紫色漸層 + 千篇一律 card grid + "Get Started" CTA）會持續拖累使用體驗。

**How to apply:**
- Bridge / 任何 web UI 出手前，先定 aesthetic direction，不要直接套 Tailwind + shadcn default
- 拒絕 AI slop 訊號（Inter、Roboto、紫漸層、grid-cols-3/4、通用 CTA 字眼）
- 產品內應有一致的 aesthetic system（tokens + typography + motion），但每個 surface 在系統內要 intentional，不 cookie-cutter
- 對外產出（文章、vault 頁面、社群）的 template 層也算 UI — typography、留白、標題層級都要設計過，不只內容對
- 評估前端方案時，「賞心悅目」是 hard requirement 不是 nice-to-have
