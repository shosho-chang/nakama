type: feedback
---

**規則**：當交付物是「人類最終要消費的產物」（字幕、影片、報告、聲音），所有自動化檢查通過只是必要條件，**實際 consume 驗收**才是充分條件。別在結構統計全綠時宣告任務完成。

**Why**：PR #109 retime 修完：unit tests 19 過、全測 1139 過、ruff 乾淨、結構不變量全 0（零時長/同起/重疊/倒退）、anchor 偏差 ≤ 3s。我報告「Goal 1 done」——結果用戶拉進 Premiere 播一秒就「一開始就漏字、對不齊」。所有自動化訊號都在騙人：匹配率 81.8%，但漏匹配的是**開頭那幾句最明顯的字**；anchor 是選好的樣本，沒代表整體感受。

**How to apply**：
- 字幕/轉錄任務：自己寫腳本抽幾個片段模擬播放（對齊音訊+字幕打印同一時間窗的內容），或 demand 用戶在真實播放環境驗一段才算完工。
- 影片/圖片輸出：screenshot 比對、UI 在瀏覽器實際操作，不是看 HTML DOM 結構就算完成。
- 報告/文件輸出：實際讀過一次，不是看字數/段落數分佈。
- 回報格式：結構統計當 sanity check（「看起來沒爛」），加一句「但需要你實際 X 驗證」。別把結構綠當成功。
- 如果用戶回「還是不行」：先停下來，別立刻再迭代演算法——根本假設可能就錯。
