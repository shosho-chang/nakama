**Date**: 2026-05-06
**Author**: sandcastle agent

## Objective

Validate ADR-020 S6 design decisions before full integration:
1. BGE-M3 (1024d) vs potion-base-8M (256d) — retrieval precision lift
2. bge-reranker-large cross-encoder — top-20→top-5 precision
3. Parent-Child chunking — small-to-big retrieval pattern

---

## Compute tier

| Component | Model | Params | Dims | Inference |
|-----------|-------|--------|------|-----------|
| Dense encoder | BAAI/bge-m3 | 570M | 1024 | GPU FP16 recommended; CPU viable for batch ingest |
| Cross-encoder reranker | BAAI/bge-reranker-large | 560M | — | GPU FP16; CPU ~200ms/pair |
| Legacy encoder | minishlab/potion-base-8M | 8M | 256 | CPU-only, <5ms/query |

BGE-M3 is ~70× larger than potion-base-8M. Ingest throughput expectation:
- Batch of 100 paragraphs: ~12s on CPU (FP32), ~3s on GPU FP16
- Query time: <100ms single-query on GPU FP16

## Evaluation structure (planned)

### Dataset
- Source: 2 chapters from `bse-2024` (Chapter 1 + Chapter 3, ~180 paragraphs each)
- Ground truth: 30 query–answer pairs annotated by domain expert (bioenergetics)
- Metric: MRR@5, Recall@10

### Retrieval pipeline comparison

| Pipeline | Dense model | Reranker | Expected MRR@5 |
|----------|-------------|----------|----------------|
| Baseline (legacy) | potion-base-8M 256d | none | ~0.52 (historical) |
| S6 dense only | BGE-M3 1024d | none | ~0.68 (projected) |
| S6 + reranker | BGE-M3 1024d | bge-reranker-large | ~0.74 (projected) |
| S6 + reranker + P/C | BGE-M3 1024d | bge-reranker-large | ~0.79 (projected) |

Projections based on BEIR benchmark transfer and bilingual (zh/en) query mix.

### Parent-Child design rationale

- **Child chunks** (window=4 paragraphs, overlap=1): dense retrieval target — small, semantically focused
- **Parent chunk** (heading + mermaid + wikilinks): pulled into LLM context on retrieval hit — provides section scope without verbatim body noise
- Expected: precision↑ vs full-section embeddings; recall maintained via overlap

### Bilingual query handling

BGE-M3 is a cross-lingual model (100+ languages). Mixed zh/en queries (e.g.
「ATP-PCr 系統的能量補充速度」) are expected to retrieve English-body chunks
without translation preprocessing. This is a key motivation for the upgrade
from monolingual potion-base-8M.
